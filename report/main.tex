\section{Objective}
% ─────────────────────────────────────────────────────────────────────────────

This laboratory delivers a production-oriented Single Page Application (SPA)
for the BluePrints ecosystem, modernizing the original static client into a
React architecture with robust state management, JWT-aware HTTP communication,
canvas interaction, and test-driven quality controls.

\begin{conceptbox}[title={Primary Learning Objectives}]
\begin{enumerate}
  \item Build a modular React UI with reusable components and route-based views.
  \item Implement global state using Redux Toolkit slices and async thunks.
  \item Integrate JWT authentication with Axios interceptors and protected routes.
  \item Support author lookup, blueprint visualization, point editing, and persistence.
  \item Provide service abstraction (real API vs in-memory mock) using environment flags.
  \item Achieve high automated test confidence with Vitest + Testing Library.
\end{enumerate}
\end{conceptbox}

\section{Architecture and Data Flow}
% ─────────────────────────────────────────────────────────────────────────────

\subsection{Frontend Architecture}

The application adopts a layered frontend architecture:

\begin{enumerate}
  \item \textbf{Presentation Layer:} Pages and reusable components.
  \item \textbf{State Layer:} Redux slice with async thunks and reducers.
  \item \textbf{Service Layer:} API adapter that normalizes backend response envelopes.
  \item \textbf{Transport Layer:} Axios instance with JWT injection and auth handling.
\end{enumerate}

\begin{warningbox}
The backend security API returns envelope payloads
(for example \texttt{\{code, message, data\}}). The frontend service
normalizes these responses before they reach Redux. This decision avoids
duplicating response-shape checks in components and reducers.
\end{warningbox}

\subsection{JWT Authentication Flow}

\begin{enumerate}
  \item User signs in at \texttt{/login}.
  \item UI posts credentials to \texttt{/auth/login}.
  \item \texttt{access\_token} is stored in \texttt{localStorage}.
  \item Axios interceptor appends \texttt{Authorization: Bearer <token>}.
  \item Protected routes become accessible through \texttt{PrivateRoute}.
\end{enumerate}

\begin{lstlisting}[style=jscode, caption={Axios interceptor strategy (simplified)}]
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})
\end{lstlisting}

\section{Blueprint Create/Update Reliability Improvements}
% ─────────────────────────────────────────────────────────────────────────────

During integration against the secured Java backend, key reliability fixes were
implemented to align frontend write operations with the backend contract.

\begin{resultbox}[title={Implemented Stability Fixes}]
\begin{itemize}
  \item Added response-envelope unwrapping in service layer to support
	  \texttt{\{code, message, data\}} and \texttt{\{status, message, data\}}.
  \item Updated save flow to persist only newly created points through the
	  backend endpoint \texttt{/blueprints/\{author\}/\{name\}/points}.
  \item Added duplicate prevention and clearer conflict feedback for blueprint creation.
  \item Added defensive point JSON parsing/normalization in creation form.
  \item Expanded regression tests for service and page-level create/save flows.
\end{itemize}
\end{resultbox}

\section{Synthesis of Recommended Readings}
% ─────────────────────────────────────────────────────────────────────────────

This section summarizes actionable lessons extracted from the requested
references and their direct application in this project.

\subsection{React Best Practices (freeCodeCamp)}

\begin{itemize}
  \item Keep components focused and reusable.
  \item Organize by feature and concern, not by file extension only.
  \item Treat side effects explicitly and isolate API calls in dedicated layers.
  \item Avoid ad-hoc global mutable state.
\end{itemize}

\textbf{Applied in this lab:} separate folders for components/pages/services,
clear route boundaries, isolated API service modules, and reusable UI blocks.

\subsection{State Management in React (freeCodeCamp + Redux Toolkit article)}

\begin{itemize}
  \item Escalate from local state to centralized state only when needed.
  \item Use predictable reducers and immutable updates.
  \item Use async abstraction patterns for network-dependent transitions.
\end{itemize}

\textbf{Applied in this lab:} \texttt{createSlice} + \texttt{createAsyncThunk},
single source of truth for blueprint state, and structured loading/error flows.

\subsection{Axios Interceptors Documentation}

\begin{itemize}
  \item Interceptors are ideal for cross-cutting concerns (auth headers,
	  centralized response handling).
  \item Keep auth token injection close to transport concerns.
\end{itemize}

\textbf{Applied in this lab:} request interceptor appends Bearer JWT globally;
response interceptor handles invalid-session scenarios.

\section{Conclusions}
% ─────────────────────────────────────────────────────────────────────────────

\begin{resultbox}[title={Key Findings}]
\begin{enumerate}
  \item The migration from a static client to a React SPA significantly improves
	  maintainability, traceability, and extensibility.
  \item Redux Toolkit provides a balanced state model for medium-size frontend
	  systems where asynchronous workflows dominate.
  \item Service abstraction plus response normalization is critical when backend
	  response contracts evolve during iterative labs.
  \item JWT integration with Axios interceptors and protected routes creates a
	  clean and scalable authentication boundary in the UI.
  \item High automated test coverage helped detect regressions early while
	  implementing advanced requirements and UX refinements.
\end{enumerate}
\end{resultbox}

\begin{note}
Future extension opportunities include RTK Query migration for request caching,
MSW-based network simulation for deterministic test isolation, and CI pipelines
that publish coverage artifacts for each pull request.
\end{note}
